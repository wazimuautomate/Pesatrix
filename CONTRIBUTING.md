# Contributing

## Change Workflow

Use this workflow for every code change in this repository:

1. Update `Changelog.md` in the same change set.
2. Create a standard non-interactive Git commit with a clear message.
3. Push the commit to `origin` after the change is verified.

## Commit Style

- Prefer standard commit messages in the form `type: short summary`.
- Use concise summaries that describe the user-visible or engineering outcome.
- Examples:
  - `fix: correct task availability query`
  - `feat: add pasted JSON import for admin tasks`
  - `docs: add contributor workflow requirements`
